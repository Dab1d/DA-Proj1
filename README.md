# Project Documentation

## Quick Start
To get started with the project, follow these steps:
1. Clone the repository: `git clone https://github.com/Dab1d/Conference-Assignment.git`
2. Navigate to the project directory: `cd Conference-Assignment`
3. Install the necessary dependencies: `npm install`
4. Start the application: `npm start`

## Architecture
The project follows a modular architecture, allowing for easy maintenance and scalability. The components are organized into the following layers:
- Presentation Layer: Handles user interaction.
- Business Logic Layer: Processes inputs and coordinates responses.
- Data Access Layer: Interfaces with the database.

## CSV Format
The data is stored in CSV format. Below is the structure of each CSV file used in the project:
- `data.csv`: Contains the main data entries with fields such as `id`, `name`, `date`, and `location`.

## Workflow
The workflow of the project consists of:
1. User inputs data via the front end.
2. The data is processed in the backend.
3. The processed data is stored in the database.
4. Users can retrieve and manipulate the data as needed.
